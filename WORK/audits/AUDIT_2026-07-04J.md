## Six real product bugs fixed (spec-only fixes could not have gone green)

1. **Hardcoded `reg_01` register default** (`web/lib/useFinderContext.tsx`) — the id
   only exists in mock data; real tenants get generated ids, stranding the terminal on
   "register 'reg_01' not found". The context now reconciles the persisted selection
   against `GET /api/v1/outlets` and adopts the first real register. (Not gated on the
   user profile — that lives in sessionStorage and is empty after a browser restart.)
2. **snake_case/camelCase wire drift → $NaN money** (`web/lib/normalizeTerminalProduct.ts`,
   applied in ProductGrid, TerminalInner, TenderScreen) — the mock layer served
   camelCase (later dual-shape) products/orders/payments; the real backend serves
   canonical snake_case rows. The terminal rendered $NaN prices and totals and the
   tender screen was unusable. All three shapes now normalize at the fetch boundary.
3. **Session-killing `silentRefresh` race** (`web/lib/auth.ts`) — refresh tokens are
   single-use; two concurrent refreshes (useAuth boot + any 401 retry) replayed the
   same cookie and the loser's catch ran `clearSession()`, wiping the session the
   winner just established. Would randomly log out real users after a browser restart.
   Now single-flight across all callers.
4. **Register guard strands cashier on 409** (`RegisterSessionGuard.tsx`) — a transient
   sessions-load failure showed the "Register closed" card even when a session was
   open; opening then failed "already has an open session" with no way forward. The
   guard now re-syncs to the live session on 409.
5. **No accessible page headings** (`EnterpriseShell.tsx`) — the shell accepted
   `title` but never rendered it; no page had an h1. Added an sr-only h1 (screen
   readers + tests; zero visual change). Also `aria-label`/`aria-haspopup`/
   `aria-expanded` on the user-menu trigger and label associations on the register
   open form (real WCAG fixes; the logout spec depends on the first).
6. **Playwright setup unable to bootstrap on fresh checkout** (`playwright.config.ts`)
   — `storageState: undefined` inherits (not clears) the global value → ENOENT.
   (Antigravity team landed the identical fix in parallel; merged theirs.)

## E2E infrastructure fixes

- **`web/e2e/fixtures.ts` (new)** — worker-scoped browser context: the backend's
  single-use refresh rotation is incompatible with Playwright's context-per-test
  snapshot replay. One context per worker behaves like a real browser session.
  Self-heals on worker restart (a failure discards the worker and the snapshot cookie
  is already rotated): probes the session and logs in fresh via the UI if dead.
  Pages drain in-flight requests before closing so a mid-rotation kill can't strand
  the shared cookie jar on a revoked token.
- **checkout.spec.ts** — opens the register (float $100) when the real backend
  presents a closed one, resilient to the register-guard remount race; cash input
  targeted via `spinbutton` role (`getByLabel(/cash/i)` also matches the Cash
  tabpanel); submit matcher includes "Collect Cash".
- **seed-e2e.ts** — products now stocked (100 on hand), plus one supplier and an
  open 2-line PO so purchasing/receiving specs exercise real rows. Also documents the
  `@name`-placeholder trap: string literals containing `@` (emails) must be bound
  parameters.

## Auth architecture finding (affects real users, not just tests)

The backend's strict single-use refresh rotation is racy against the browser page
lifecycle: when a page unloads mid-rotation, the server has already revoked the old
token but the Set-Cookie carrying its replacement never lands — the user is
permanently logged out. Playwright reproduced this reliably; real users hit it on
fast navigations/tab closes after a browser restart. Fixes, layered:

- **Rotation grace window** (`src/identity/service.ts`): a rotated token stays
  acceptable for `REFRESH_REUSE_GRACE_MS` (default 15 s, set 0 for strict) after
  revocation; a graced replay issues a fresh pair without re-revoking. Replays
  outside the window still 401 — theft/reuse detection intact. This is the standard
  "reuse interval" pattern.
- **Client single-flight** (`web/lib/auth.ts`, bug #3 above) removes the app's own
  concurrent-refresh self-logout.
- **e2e drains** (fixtures, global.setup) let in-flight rotations settle before
  pages close; the worker fixture self-heals via UI login if a snapshot cookie died.
