## SEC-4 fix

`web/app/(protected)/catalog/_components/PrintLabelsModal.tsx` previously built the
popup print document from an HTML template string containing product fields. The fields
were escaped, but the safer pattern for user-controlled product data is to avoid HTML
string interpolation entirely.

Changed behavior:

- Print popup document is assembled with DOM APIs.
- Product `name`, `sku`, and `barcode` are assigned with `textContent`.
- The print/close behavior still runs after the popup document is populated.

Regression test:

- `web/tests/printLabelsModal.test.tsx`
- Uses malicious product fields containing attempted `img`, `script`, and `svg` markup.
- Verifies those values render as text and do not create executable DOM nodes.
