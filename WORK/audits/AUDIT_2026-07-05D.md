## Scope

Production post-deploy smoke alignment after the backend operational readiness deploy.

Claim: Codex session N in `WORK/LOCK.md`.

## Finding

The production deploy for session M succeeded, but the CI post-deploy smoke job failed
at `GET https://finder-pos-backend.vercel.app/api/v1/flags`.

This was not a backend regression. The live backend returned `401 unauthenticated`,
which is the intended protected-route behavior and is exactly what `npm run ops:check`
verifies as the auth boundary. The smoke workflow was stale: it used `curl -f` and
therefore expected unauthenticated `/api/v1/flags` to return public `200`.

## Change

`.github/workflows/ci.yml` now checks that unauthenticated `/api/v1/flags` returns
`401` during post-deploy smoke.

## Verification

- Live production `/api/v1/flags` auth-boundary curl: PASS (`401`).
- `git diff --check`: PASS.

## Remaining truth
