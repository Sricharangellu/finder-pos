## Why this happened

The latest CI run after the backend operational-readiness work passed the backend and
production gates, then failed only in Playwright:

- Production guard: PASS
- Backend typecheck + tests: PASS
- Real-Postgres smoke: PASS, including the expanded register open -> close -> EOD
  segment
- Frontend typecheck/lint/build: PASS
- Production deploy: PASS
- Production post-deploy smoke: PASS (`/healthz`, `/readyz`, flags auth boundary,
  frontend)
- Playwright E2E: FAIL

The failing Playwright specs were stale relative to the current product rule. The old
`web/e2e/verticals.spec.ts` still said CI activates all modules and expected full
create/action flows for appointments, healthcare, automotive, manufacturing, rental,
entertainment, education, and module-marketplace/onboarding surfaces.

That contradicts `WORK/RULES.md`: Ascend is a modular business platform, but only
**Retail** may be treated as the current Built/verified release pack. Non-retail packs
are Preview until retail is complete end-to-end.

## What changed

- Rewrote `web/e2e/verticals.spec.ts` as retail-first business-pack coverage.
- Preview vertical routes now verify authenticated rendering and absence of unhandled
  crash/error boundary, instead of claiming full production workflow readiness.
- Module Marketplace tests remain because that is part of the business-pack control
  plane, but assertions avoid brittle duplicate-heading strict-mode traps.
- Onboarding now asserts the honest `Ready` / `Preview` distinction, matching the
  latest onboarding implementation.
- Widened `web/e2e/inventory-receive.spec.ts` receive-stock text matching to the
  current page copy (`Select pending PO`, `Choose a PO to receive`, `Scan barcode`).
