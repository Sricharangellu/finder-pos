# Audit — 2026-07-03B (deep verification: live stack, brutal truth)

Immutable snapshot. Second audit of 2026-07-03 (separate session from AUDIT_2026-07-03.md).
Method: booted the REAL stack — local Postgres 15, backend on :3001, migrations + e2e seed,
authenticated HTTP probes against every frontend-declared endpoint, smoke lifecycle,
Playwright attempt. Evidence only; no claims from docs.

## Headline

**The backend core is real and works. The deployed frontend is a mock demo by explicit
config. The orchestration layer is dead code against a real database. Both facts were
invisible because every prior "passing" signal came from stubs.**

## Verified WORKING (real Postgres, real HTTP — label: Built and verified)

- Fresh-database migrations complete cleanly; 46 modules mount.
- `npm run smoke`: **13/13 steps** — login (tenant `tnt_demo`, owner role), product create
  with tax classes, stock receive, order with correct CA tax ($3.30 on $50, exempt item
  honored), split payment (cash + simulated card), automatic inventory decrement,
  offline outbox (10 events queued offline → all synced on reconnect), refund with
  restock to original level, Prometheus metrics endpoint.
- Endpoint probe: extracted all **484** endpoints the frontend mock layer defines and hit
  each on the live backend with a real JWT: **~464 exist** (non-404). Customer-360
  detail/patch/loyalty/summary/financials all return 200 with real entities. Catalog
  detail 200. The "frontend is mostly unbacked mocks" fear is FALSE at the API level.

## Verified BROKEN (label: Not production-ready / defects)

1. **Orchestration layer fails on a real database — silently.** `workflow_instances` and
   `saga_instances` tables are used by `src/orchestration/state/*.store.ts` but NO
   migration creates them. Every workflow trigger during smoke (checkout,
   order_fulfillment, refund) threw `relation "workflow_instances" does not exist` and
   the flows only succeeded because module services do the work inline. Additionally the
   command registry logs **21 unregistered commands** at boot (checkout.validate_cart,
   reserve_inventory, authorize_payment, payment.process_refund, accounting.post_journal…).
   The 44 orchestration unit tests pass because they stub the DB. **The celebrated
   workflow/saga layer is currently decorative in production.**
2. **Production frontend is 100% mock mode by hardcoded config.** `web/next.config.mjs`
   set `NEXT_PUBLIC_MOCK: "true"` unconditionally, comment: "there is no real backend in
   this deployment." Deployed builds never call the real API. Fixed this session to be
   env-overridable (`NEXT_PUBLIC_MOCK ?? "true"` — default behavior unchanged); flipping
   the default is a deliberate Phase-2 decision.
3. **The documented local e2e procedure cannot work.** `NEXT_PUBLIC_E2E_MODE` is read by
   nothing in the app (only playwright.config references it); dev server ALWAYS starts
   MSW (`NODE_ENV === "development"` short-circuit in `MockWorkerInit`), so "e2e against
   real backend" via `npm run dev` actually tests mocks — or hangs on the init spinner.
   e2e README also lists stale credentials (`owner@demo.com` vs seeded
   `owner@finder-pos.dev`).
4. **All 46 Playwright tests were blocked by one stale locator** — setup waited for a
   `finder` heading the redesigned login page no longer has. Fixed
   (`sign in` heading). Production-build e2e run: see results section below.
5. **8 stale vitest tests** (`catalogCart.test.tsx` 5, `reportsDashboard.test.tsx` 3) —
   components drifted; tests not updated. Still open.
6. **Mock-only endpoints include CORE inventory operations.** Missing from the backend
   (frontend works only via MSW): `POST /inventory/transfers`, `POST /inventory/adjustments`,
   `POST /team` (invite), `GET /workflows/templates`, `POST /reports/ar-aging/sweep`,
   `GET /team/:id`, and the entire Vendor-360 family
   (`/purchasing/vendors/:id` + `/products|purchase-orders|invoices|credits|receiving` —
   backend only has `/purchasing/suppliers`). ~12–14 of 484 endpoints are mock-only.
7. **RLS is a safety net with holes, not enforcement.** Policies are enabled + FORCED on
   all tenant tables (good), but deliberately permissive when `app.tenant_id` is unset,
   and `db.withTenant()` is adopted in only ~10 of 46 modules (57 call sites). Most
   queries rely purely on hand-written `WHERE tenant_id=` clauses. A single forgotten
   WHERE clause in the other ~36 modules = cross-tenant data leak with no RLS backstop.
8. **Repo/workspace hygiene.** Tracked duplicate `src/shared/db 2.ts` (stale copy of
   db.ts, imported by nothing) — removed this session. **25 stale agent worktrees** under
   `.claude/worktrees`: 9 clean+merged removed this session; **15 remain** (dirty and/or
   on 10 unmerged branches) — need manual review before deletion, they may hold lost work.
9. **Agent instructions did not exist.** No `AGENTS.md`/`CLAUDE.md` in the repo;
   workspace `Agents/AGENTS.md` was an empty stub. Fixed this session: repo `AGENTS.md`
   created, workspace file rewritten; both enforce the WORK/ single-folder rule.

## Production-build e2e (mocks off, real backend) — FIRST COMPLETE RUN EVER

Setup: `NEXT_PUBLIC_MOCK=false npm run build` → standalone server on :3000 proxying to
the real backend on :3001 (local Postgres 15, e2e seed). Two stale locators fixed to
unblock the suite (login heading, password field vs show-password toggle).

**Result: 25 passed / 22 failed (6.0 min, 47 tests).** This suite had NEVER completed
before — it was broken at the setup step, so no e2e signal has existed for any commit.

What the run proves:
- Real browser login against the real backend works; the authenticated shell renders
  with seeded tenant data (register `reg_01`, Main Outlet).
- 25 golden-path assertions pass with zero mocks.

Failures (need per-spec triage; two classes observed):
- **Core flow, 10 failed**: checkout ×3, inventory-receive ×3, invoice-pay ×3, logout ×1.
  Confirmed at least partly stale locators (e.g. terminal search placeholder changed);
  some may be pages that only work with mock data.
- **Vertical pages, 12 failed**: appointments, healthcare, automotive, hospitality ×2,
  manufacturing, rental, entertainment ×2, education ×2, module marketplace. These pages
  were built against mocks; against the real backend several fail the "no error boundary"
  check — i.e. some vertical pages crash without mocks. Consistent with the expansion
  pause: this breadth is UI-only/Mocked, not integrated.

Release-gate status for "Playwright golden paths pass": ❌ FAILING (25/47), now honestly
measurable for the first time.

## Corrections to previous audits (evidence beats narrative)

  (mock flag) and ~14 missing endpoints, not a hollow backend.

  core spine end-to-end and now runs green locally. It belongs in CI.
