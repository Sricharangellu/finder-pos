## SEC-7 finding

`src/identity/routes.ts` already defines the shared auth cookie options with:

- `sameSite: "lax"`
- `path: "/"`
- `secure: true` only when `NODE_ENV === "production"`

`setAuthCookies()` applies those options to:

- `finder_refresh` with `httpOnly: true`
- `finder_session_hint` with `httpOnly: false`

So the original SEC-7 concern was stale: the refresh cookie already has SameSite
protection. The missing piece was route-level proof.

## SEC-7 fix

Added backend regression coverage in `src/identity/identity.test.ts`:

- Performs a real `POST /api/identity/login` against the Express app.
- Reads the actual `Set-Cookie` response headers.
- Asserts `finder_refresh` includes `HttpOnly`, `SameSite=Lax`, and `Path=/`.
- Asserts `finder_session_hint` remains JavaScript-readable but also uses
  `SameSite=Lax`.

Updated `src/identity/test-request.ts` to expose response headers to tests while
preserving existing `status`/`json` callers.
