## Headline

Retail-first queue item #2 is Built and verified.

Finder now has a read-only backend impact contract for previewing business-type and
module changes before applying them to a tenant. This is the backend proof needed before
setup, settings, or demo switcher UI starts claiming that business-mode switching is
safe.

## What changed

- Added `SettingsService.getCapabilitiesImpact(auth, target)`.
- Added top-level `GET /api/v1/capabilities/impact`.
- Added settings alias `GET /api/v1/settings/capabilities/impact`.
- Added query parsing for:
  - `businessType`
  - `enabledModules`
  - `disabledModules`
- Impact response now reports:
  - `impactVersion` and `readOnly`;
  - current and target business type summaries;
  - whether business type changes;
  - added, removed, unchanged, and target-enabled modules;
  - required-field deltas by entity;
  - workflow deltas;
  - permission, report, and page deltas derived from changed modules;
  - setup tasks required by the target business type;
  - slim current and target capability snapshots;
  - the exact apply payload for `POST /api/v1/settings/business-profile`;
  - warnings for read-only behavior, unfinished entitlement enforcement, and the
    retail-first release rule.
- Previewing a business-type change ignores existing stored module overrides so the
  tenant can see the target pack defaults clearly.
- Previewing only module overrides keeps the current business type and applies the
  requested enable/disable diff without mutating stored settings.
- Invalid business types, unknown modules, disabling core modules, and contradictory
  enable/disable requests are rejected with `400 bad_request`.

## Honest limitations

- This is still backend-only. No setup/settings/demo UI consumes the endpoint yet.
- Plan-to-module entitlement enforcement is still not built.
- Required fields and workflows are impact-contract data, not full database-enforced
  validation across every module.
- Permissions and reports in the preview are registry-derived deltas; endpoint-level
  RBAC enforcement still needs the later audit/implementation pass.
- Only retail may be treated as Built and verified until retail release gates pass.
