## What shipped (`7cca4df`)

Two plan requirements from FORWARD_PLAN "Required setup, auth, settings, and demo UX":

### 1. "Setup must show required next tasks for retail"

New `web/components/setup/RetailSetupChecklist.tsx` — the seven required tasks
(outlet, register, tax rate, payment modes, receipt, first product, first receiving)
with **live completion detection**: each check queries the same API the corresponding
setup page writes to (`/outlets`, `/settings/tax-rates`, `/settings/payment-modes`,
`/settings/receipts/:outletId`, `/catalog`, `/inventory/levels`), so the card reflects
real tenant state on both mock and real backends rather than a stored wizard step.
Detection fails closed — an unreachable backend shows tasks as not-done instead of
falsely complete. Mounted on the dashboard for retail tenants: progress bar, deep
links to each setup page, dismissible per browser, auto-hides when everything passes
or the tenant isn't retail.

### 2. "Setup must not present every vertical as equally complete"

The onboarding wizard's business-type step now renders from the capabilities registry
(`availableBusinessTypes`) with the old static list kept only as fail-open fallback and
icon enrichment. Retail sorts first and is badged **Ready**; every other type is badged
**Preview** with a legend explaining the difference, and confirming a Preview type shows
an honest amber notice (pack activates now; the complete end-to-end workflow is
verified for Retail first; shared features work today). This matches the demo-switcher
honesty rule: only retail may be presented as Built and verified until other packs pass
their own gates.

- The mock demo tenant intentionally has no tax rates or payment modes seeded, so in
  demo mode the checklist genuinely guides those steps — good demo behavior.
- Receipt detection treats any saved header/contact/footer text as configured; the
  backend returns defaults when unset, so an explicitly saved empty template still
  reads not-done (acceptable: nudges configuration).
- Remaining "Signup and setup" plan gap: the signup page's business-type grid is
