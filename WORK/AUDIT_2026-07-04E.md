## SEC-8 fix

`web/app/(protected)/imports-exports/page.tsx` previously exported catalog CSV with:

- a direct `fetch()`
- manual `Authorization: Bearer <token>` header
- no shared 401 refresh/retry behavior
- no API error envelope parsing

Changed behavior:

- Added `apiDownload()` to `web/api-client/client.ts`.
- `apiDownload()` attaches the bearer token through the shared API-client path.
- On one 401 response, it performs the same silent refresh and retry flow as `apiFetch`.
- Non-OK responses surface `ApiResponseError`, including JSON API error envelopes when
  present.
- Catalog export now calls `apiDownload("/api/v1/catalog/export")`.

Regression tests:

- `web/tests/api-client.test.ts`
- Proves blob downloads include the bearer token.
- Proves a 401 export response triggers silent refresh and retries with the refreshed
  token.
